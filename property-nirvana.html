Project Overview

Finding your home
should feel natural.

Property Nirvana is a real estate platform for property discovery and agent connection. The platform had grown organically, resulting in a cluttered interface with an overwhelming filter system and property cards dense with information users didn't need at the browsing stage.

Users were spending too long browsing without converting to agent contact — the path from discovery to enquiry had too many friction points and unclear next steps.

My task was to redesign the core discovery experience — simplifying filters, restructuring property cards, and creating a clear, progressive disclosure path from browsing to contacting an agent.

Challenge & Solution

The discovery problem

🔴 The Problem

Cluttered interface, lost users

The existing filter system exposed 20+ parameters upfront, overwhelming first-time visitors. Property cards showed all available data at once — price, sq ft, agent name, listing date, floor plan — creating cognitive overload. The path to contacting an agent required 6+ steps with no visual momentum.

🟢 The Solution

Progressive disclosure + clear hierarchy

I reorganised filters into a tiered system (Essential → Advanced), showing only 5 key filters upfront with a contextual "More Filters" expansion. Property cards were rebuilt with a strict 3-tier hierarchy: image → price + location → key specs. A persistent "Contact Agent" CTA was introduced on every card.

🔍 Research

Competitive benchmarking + IA audit

Analysed 5 real estate platforms (Rightmove, Zoopla, Trulia, Zillow, 99acres) to understand filter conventions and card patterns. Conducted an information architecture audit to identify which data points users actually need at each stage of the decision journey.

✅ Outcome

A focused, conversion-driven experience

The redesigned discovery flow reduced steps to agent contact from 6 to under 3. The new property card design improved information scannability, and the simplified filter system made narrowing down options feel guided rather than overwhelming.

Design Decisions

Key design decisions

🗂️

Tiered Filter System

Essential filters shown by default, advanced filters hidden behind a contextual toggle — reducing initial complexity without sacrificing power users.

🏠

Restructured Property Cards

Three-tier visual hierarchy on every card: hero image → price + location → secondary specs. Critical info first, detail on demand.

📍

Map-First Toggle

Introduced a prominent map/list toggle at the top of results so location-focused users could switch to a spatial view immediately.

📞

Persistent Agent CTA

Contact agent button visible on every card and sticky on the detail page — removing the need to hunt for a way to get in touch.

💾

Save & Compare

Redesigned the save flow and introduced a side-by-side comparison view so users could evaluate shortlisted properties without context-switching.

📱

Mobile Responsive

All layouts designed mobile-first, with bottom-sheet filter panels and swipe-friendly gallery interactions on property detail pages.

Key Learnings

What this project taught me

01

Progressive disclosure is a superpower. Showing everything upfront feels helpful but actually overwhelms. Tiering information by decision stage — and trusting users to ask for more — dramatically reduces cognitive load without sacrificing functionality.

02

High-stakes decisions need trust signals. Property browsing is a high-stakes, high-anxiety activity. Small UX details — agent photos, response time indicators, verified listing badges — had an outsized impact on making the interface feel trustworthy.

03

The goal isn't browsing — it's converting. Every design decision should be evaluated against whether it helps users take the next real-world step (contacting an agent, booking a viewing). Beautiful but passive discovery experiences don't serve anyone.
